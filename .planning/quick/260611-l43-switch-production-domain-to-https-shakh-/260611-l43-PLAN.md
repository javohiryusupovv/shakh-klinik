# Quick Task 260611-l43: Switch production domain to shakh-clinics.ru

## Goal

The clinic site has moved to its production domain **https://shakh-clinics.ru**.
Update the single source of truth for the site URL so that every SEO and
social-preview surface (canonical URLs, Open Graph tags, Twitter/Telegram
cards, robots.ts, sitemap.ts, `metadataBase`) resolves to the new domain.

Open Graph is what drives rich link previews on **Telegram, Facebook, and
WhatsApp** — all three read `og:title` / `og:description` / `og:image` /
`og:url`. The metadata pipeline in `lib/seo/metadata.ts` already emits these
correctly; the only thing pointing at the old domain is `SITE_URL`.

## Scope boundary

- IN: `NEXT_PUBLIC_SITE_URL` env value + the hardcoded fallback default.
- OUT: The contact email `info@shaxklinika.uz` and the `"ShaxKlinika"` brand
  name — those are business decisions, flagged separately, NOT changed here.

## Tasks

### Task 1 — Point SITE_URL at the new production domain

- **files:** `.env.local`, `.env.example`, `lib/seo/config.ts`
- **action:**
  - In `.env.local`: change `NEXT_PUBLIC_SITE_URL=https://shaxklinika.uz` →
    `NEXT_PUBLIC_SITE_URL=https://shakh-clinics.ru`
  - In `.env.example`: change the same line and any explanatory comment that
    names the old domain.
  - In `lib/seo/config.ts:4`: change the fallback
    `?? 'https://shaxklinika.uz'` → `?? 'https://shakh-clinics.ru'`
- **verify:**
  - `grep -rn "shaxklinika.uz" lib/seo .env.local .env.example` returns no
    matches.
  - `grep -n "shakh-clinics.ru" lib/seo/config.ts .env.local .env.example`
    shows the new domain in all three.
- **done:** All three files reference `https://shakh-clinics.ru`; the contact
  email and brand name are untouched.

**Truths:**
- `SITE_URL` resolves to `https://shakh-clinics.ru` in production.
- canonical, OG, Twitter/Telegram, robots, sitemap, and metadataBase all derive

**Artifacts:**
- `lib/seo/config.ts` fallback updated.
- `.env.local` and `.env.example` updated.

**Key links:**
